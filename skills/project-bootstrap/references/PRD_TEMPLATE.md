# PRD Template

Use this as the default shape for `PRD.md`.

```md
# PRD

Status: Draft
Date: [YYYY-MM-DD]
Owner: [name or self]

## Title

## Problem

## Target Users

## Goals

## Non-Goals

## Current Phase

## Scope For This Phase

## Success Criteria

## Product Constraints

## Technical Constraints

## Architecture Considerations

## First Milestone Candidate

## Open Questions

## Approval Notes
```

Approval rule:

- only change the status to `Status: Approved` after explicit user approval
- if the user asks for revisions, move back to `Status: Needs Review`
